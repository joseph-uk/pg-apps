3d in flyxc is now live.

You can check an example flight.

I am really excited about it. FlyXC (or VisuGPS as it was named before) used to support 3D based on the google earth plugin then DoArama both the former has been discontinued and the latter now Ayvri has become a paid platform. FlyXC now uses the great ArcGIS API from ESRI.

The navigation in 3D can (and will) be improved but I think the current state could still be valuable to plan flights and see your tracks.

For now 3D supports the same navigation as 2D:
- click on the graph to center the track on the current location,
- scroll the mouse wheel on the graph to zoom on the current location.

If you have any suggestions for improvement you can comment here or on github if you have an account.

Many changes have been done for this 3D version. Please report is anything is no more working as expected.

There are many more improvements coming to flyxc (i.e. as I can find time to work on it). FlyXC has been open source since the beginning (~13 years). You can contribute to the source code if you know at least at little about development. I would also love to have people contributing a better documentation as the "README" is both too short and outdated.

VictorB wrote:
It seems like Garmin has changed the InReach API which causes problem for flyxc.app to retrieve the InReach positions.

I'm trying to see how this can be fixed...

Stay tuned for an update


TL;DR: The situation should be back to normal with InReach devices.

Technical details:

FlyXC use the InReach API to retrieve the pilot positions. Garmin recently added a limit on how often a server can use this API (without communicating on that change/limit). I am trying to contact someone at Garmin to see if flyxc could get a higher limit. In the meantime I have implemented a workaround: when the flyxc server reaches the limit, a proxy server is created in the cloud to query the InReach API. It solves the problem while keeping the cost low as proxy servers are only started as needed.
