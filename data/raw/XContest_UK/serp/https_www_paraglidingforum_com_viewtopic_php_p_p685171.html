Hi I'm the developer of PureTrack.io, the ultimate live tracking website. I had a number of requests to add XContest and XCTrack support. This has now been done, and I'm looking for testing and feedback.

It's super quick and easy to set up, instructions here:
https://v2.puretrack.io/help/xcontest

For those not familiar PureTrack.io is a live tracking website, that combines many tracking systems into a single place. Similar to SafeSky, but works worldwide, and you can add anything to it (e.g. your crew!), not just aircraft.
See who's flying on any day, see other aircraft around you, follow your friends or watch contests. Great for crews, has an instant google maps link for directions to your pilot. And know where your mates are at all times. There's a dedicated freeflight button up top to filter out all other aircraft.

PureTrack supports:
- SPOT and InReach
- Open Glider Network i.e. OGN (FLARM/FANET)
- ADSB
- dedicated cellular trackers
- cellular apps
- SkyLines.aero
and may more!

Cheers
Tim

Aeron wrote:

Hey thanks for the feedback, yeah I've struggled to make that aspect of it less confusing. The idea is:
Multiple trackers can all point to the same 'target/label'.

And yes sounds like you got it right:
- Create a label when adding any type of tracker.
- If you add another tracker, you can select the same label (don't make a new one).
Then they all show up on the map as the same thing.

When viewing your track on the map, you can hover over the chart on the bottom to see the source of each point. So you might for example have a SPOT, the OGN and XContest all contributing to your track giving you amazing coverage!

Feel free to get in touch if you have any other issues. https://puretrack.io/contact

Aeron wrote:

1) the concept of target could be skipped, just have trackers and labels because 2)

2) to create a label (or in this relation it is a target (?) ) with the HEX code coming from OGN does work but doesn't make sense, only confusion.


Sorry yes it's confusing, but a 'target' and 'label' are the same thing. I'm going to standardise on the term 'label' everywhere, as I think that makes sense? i.e. it's a label you put on the map that can have multiple trackers feeding into it. Some other names for it I've considered:
- Pin
- Target
- "Thing" on the map"!...

Aeron wrote:
Better would be to just accept names or HEX codes (only as name) but not to link them with OGN automatically. HEX codes of OGN should only work in connection with a OGN tracker. The target concept seems to be a mixture of tracker and label.


The way it works at the moment is:
- OGN always supplies a HEX code, and sometimes an aircraft rego and contest ID.
- If it doesn't come with anything except the hex, I display the hex.
- If it comes with an aircraft rego it will show that over the hex.
- If it comes with a contest ID it will show that over the rego.
- If you manually add an "OGN" tracker, you can add your own label to a hex code, that will then display that instead of anything else.

Aeron wrote:
3) what you could do is to refuse creating a label if there is no tracker.


Generally people can only add a label if adding a tracker, so I think that's OK. You can also have a list of labels you use frequently and switch a tracker between e.g. using a SPOT for multiple aircraft.

Aeron wrote:
4) in the label view you can list all trackers and link the label to the tracker with a tick-box, not the other way round
just my 5 cent


In the dashboard you can either edit a label, and see what trackers are feeding to it (there might be multiple) OR edit a tracker and see which label it's currently using.

I have tried simplifying it a bit yesterday, by automatically creating a label with your name when new people register. And that will automatically be selected when adding an OGN tracker.

Aeron wrote:
it would be nice if the settings and filters would be linked to ones account


Yeah I should, at the moment settings are linked to each device's browser. So handy if you want different settings on your phone vs a laptop.

Thanks for the feedback, it's very helpful to think about this more deeply and to know what people are confused by.

SFparaglide wrote:

1. Have you tried to see if you can integrate with the FlySkyHy app for Iphone users?


Yes I have, I haven't heard back from them yet. In the meantime you can either send data from FlySkyHy to XContest (recommended) or Skylines. Using XContest will load all your track data. Skylines isn't quite as good in that regard, it gets your location every minute.

SFparaglide wrote:

2. I also have a skytraxx device with Fanet, should this automatically get picked up by puretrack.io through the OGN network assuming there is a Fanet to ground station or internet connection?


FANET ends up on the OGN network from what I understand, which then goes into PureTrack. It should work if either your device or someone you are connected to, has internet connection. So FANET is basically a protocol to send information between devices in air. Maybe let me know if it works? If it does, you should show up on PureTrack in your location as a random ID number. I've never used a FANET device so not super familiar with it.

SFparaglide wrote:

3. Is it possible to have a permanent link URL that we can share with contacts so that they can see our particular latest tracks?


Yes, you can almost do this now - click on any icon on the map, then click 'share' button (works in Safari at least) or get the URL out of your browser. This will give you a link to that person that will always take the recipient to that label's location. You can also find the share link for any label on the dashboard page https://puretrack.io/dashboard

Thanks for trying out PureTrack! Hope that helps others too

Hi good stuff, sorry just saw this so bit late. For the labels:

- The new 'wobble' labels have a lower limit on how many are visible before they all are hidden. You can change it under layers -> Icon Labels -> Options.
- You can also switch back to the old labels if you prefer them.
- There is also a setting to hide 'unknown' items, so if they only have random ID numbers instead of names, they will be hidden, including the second line. Turn that on if you do want second line info for unknown items.
