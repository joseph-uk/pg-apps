https://pg-race.aero/

The PG Race App is also free and without adverts and available here:
https://play.google.com/store/apps/details?id=indysoft.pgrace

Good question Bruno.
How can we be 100% confident we are live-tracking?

The first place to check is in App Status.
Click the App Status ⓘ button
In general:
Green text means normal / ok
Orange means some error

For each tracking technology, if the App says “tracking active” then you can be sure you are being tracked.


However if you also want to appear in the Pilot List and see yourself on the map, each system needs slightly different settings:


Livetrack24
===========
For Livetrack24, when you are logged in (with correct username/password), to appear in the list and on the map, in Settings, set “Track yourself into LT24” and include your Livetrack24 username in “Track other users”.

Select “Continuously track to LT24” if you are stationary and not actually flying.


SkyLines
========
Using your correct SkyLines Tracking Key, in Settings, tick the option “Track other SkyLiners” then select a distance range under “Track SkyLiners nearby” or include your own tracker name in “Track named SkyLiners”. Then you will show up in the Pilot List and on the Map. To hide yourself you have to put your name or ID into “Exclude specific SkyLine IDs”

Set “SkyLines tracking mode” to ‘continuously’ if you are not flying.


AirWhere
========
Using both your correct 6-digit Hex AirWhere pilot ID and 3-digit code, in Settings, include your 6-digit ID in “Track specific AirWhere pilots”.

Set “AirWhere tracking mode” to ‘continuously’ if you are not flying.


Open Glider Network (OGN)
=========================
Using your correct 6-digit Hex ID, which has been verified by the App, in Settings, under “Show OGN pilots nearby” set a suitable range, then check “Include your own OGN device ID”.

Set “OGN tracking mode” to ‘continuously’ if you are not flying.

However, you will not see yourself when only using OGN buddy filters, such as
XCG* => see all pilots tracking using XC Guide
FLR200* => See all pilots using XC-Tracer FLARMs
OGN* => See all pilots using OGN tracking

The extra setting “Exclude specific OGN IDs” is provided to ignore other FLARMs that you may have with you.


XC Globe
========
Using both your correct XC Globe username and password, in Settings, to appear in the list and on the map, include your name in “Track specific AirWhere pilots”.

Check “XC Globe track & query”.

Mike thanks for the great feedback.

This is now fixed in XC Guide versions V244 & upwards.
(current release is V246, Oreo Go Edition is V247).

The problem was related to new privacy rules by Google Play which require a foreground service (notification icon) to be shown while any App is taking GPS fixes.

On Android 10 however, this on it's own wasn't enough, and an additional special marking was required in the App manifest.

To further protect privacy, on Android 10 onwards, to truly get user Location "in the background" i.e. nothing showing on the screen, not even an icon, which many of us find pretty creepy, requires a new permission: ACCESS_BACKGROUND_LOCATION. This has to be applied for to Google specifically, with a significant disclosure etc and justification case, and they may or may not choose to grant the developer leave to deploy it. Fair enough.

It is only with this permission that you can get the prompt: 'Allow all the time'

Note: XC Guide does NOT use ACCESS_BACKGROUND_LOCATION

There will always be the notification icon while XC Guide is running, including when other Apps are 'on top'.

FANET is already a popular medium range free-flight communication bearer, and no doubt will be even more popular in future, as tracking, messaging and weather reports get more integrated and work between different instruments, and as more ground stations get deployed.

XC Guide V248 (beta) includes support for the new low-cost GXAirCom FANET module.

The App connects automatically via Bluetooth low energy (BLE).
The XC Guide status pop-up now shows your GXAirCom Module ID, battery level, fix status and BLE link quality.
If connected, a small FN icon appears in the flight computer box 5 (the Connectivity & Time box).

The first time XC Guide connects, your GXAirCom module MAC address is remembered, so you don't accidentally re-connect to another module on a busy takeoff. (This is the same App behaviour as linking to XC-Tracer & BlueFly varios and iTag buttons.)

Other FANET devices within range, automatically show up in your Pilot List, subject to any filters that you have set.

The FANET module name is only sent about once a minute, so the first view of any FANET pilot is only their 6 digit ID, which is sent every second. The proper module name is updated automatically in the Pilot List once it is received.

Just click on a FANET pilot to bring up the messaging options;

1) Preset messages are picked from your user-defined list from the file /XCGuide/Retrieve.txt
2) Custom messages are just typed-in, and support emojis
3) 'Need Retrieve' message & cancel Retrieve.
4) 'SOS' message & cancel SOS.

A few pilots have asked about how to get set up on OGN using XC Guide

1. Get an OGN ID

In the App Status popup window, first click the 'New ID' button.
This searches the online OGN database, finds the first available ID, and saves it in your Settings.

2. Register on OGN

The 'Reg' button takes you to http://ddb.glidernet.org/
Enter your new ID and register with same the 3-digit Competition Number (CN) provided by XC Guide. This is necessary to ensure you are the owner of the OGN Device ID. Select 'Device type' as 'OGN'.

3. Verify your ID

Once the Device ID is registered on OGN, click the 'Verify' button. A drum sounds and your Device ID is verified on XC Guide.

In OGN settings, your live tracking mode can be set (never | after take-off | continuously).

That's it - you're set up on OGN !



Advanced options

If you subsequently change your Device ID in settings, you must re-register it and re-verify.

In OGN settings, you can show OGN trackers nearby, and select to show last fixes of trackers offline.

You can choose to include your own OGN tracker in the list, and exclude specific OGN IDs.

You can also set your aircraft type.

If you want to track specific OGN Device IDs, your 'Buddies', enter each Device ID in the box called 'Show specific OGN Device IDs'.
For example XCG640000/XCG640001.

You can also use a wildcard to set filters.
For example possible OGN buddy filters are
XCG* => see all pilots tracking using XC Guide
FLR* => See all pilots using FLARM
FNT* => See all pilots using FANET
OGN* => See all pilots using OGN tracking
Separate each filter with a '/'.

In your Pilot List, these filtered OGN IDs are shown with a green 'B' on the OGN icon to indicate they are 'Buddies'.

Finally to access a different APRS server, enter your APRS URL or public static IP address in the 'Alternate APRS Server' box.

When being tracked into OGN you will appear on the following live websites:-

http://live.glidernet.org/
http://glidertracker.org/
https://planefinder.net/

Please note: Google is enforcing 'scoped storage' from this summer, when all Android Apps must target the latest Android 11, (SDK 30).

It means it is necessary to introduce a new folder structure for both the PGRace & XC Guide Apps, and the App files will be a bit harder to find.

With 'scoped storage' Apps can only access their own files in the folder:
Android.data.<package_name>.files

I believe iPhones have enforced this from the beginning, but Android is catching up.

For the PGRace App it means the feature of sharing folders for Tasks, Waypoints, Airspace and Tracks with XC Guide or XCTrack has to be removed.

Tasks as a saved QR code picture can still be easily picked as a way of loading a task across Apps.

To make the transition painless, the first time the new versions run, they will copy the relevant directories over to the new structure automatically. These transition versions still target Android 10 (SDK 29), for now, so they can 'see' the legacy folders to copy them.

Just for safety, the legacy external storage folders will be left in place, not deleted, but no longer used.

To move files such as waypoints, airspaces, tasks etc. everyone should still be able to use the built-in file picker App.

This is an advanced and fairly complicated configuration Dmytro but it can be made to work, by using XC Guide as a Local host, pushing the XC-Tracer GPS positions and Livetrack24 pilot positions into XCSoar as shown below.

Firstly, however, be aware that since Android 6 Marshmallow, Google requires Location to be enabled as well as Location Permission, before BLE Bluetooth scanning can take place. This is a privacy / security requirement of the operating system. The reason given was that BLE scanning can detect nearby Bluetooth beacons and markers which can (via a lookup) be used to identify precise locations of the phone to an App, down to a few metres. So for this reason if you disable Location, although existing BLE links will be maintained, re-scanning for an XC-Tracer which has disconnected, for example, will not work, until you re-enable Location.

To use an XC-Tracer to support Livetrack24 on XC Guide and XCSoar, set up Localhost in both Apps.

Just a few clicks:

1) Switch on XC-Tracer

2) Enable Bluetooth and Location in the Phone

3) In XC Guide settings:
a) Enable GPS from XC-Tracer
b) Enable Localhost in System settings
c) Set Localhost type as XCSoar /LK8000
d) Enable Public LT24 tracking and set to track LT24 continuously.

4) In XCSoar config:
a) Set a device as a Localhost TCP client
b) Set XCSoar port # to the XC Guide FLARM emulator port #
c) Set XCSoar Driver as FLARM

5) Check XC Guide is connected to your XC-Tracer and it has a GPS fix

6) Check Livetrack24 tracking is active.

7) Check XCSoar is connected to XC Guide Localhost, and check Livetrack 24 pilots are showing in XCSoar 'radar screen'
